## 1. プロジェクト基本方針

### 1.1 目的
セキュリティエンジニア志望の学生「KISO」のスキル、経歴、制作物を魅力的に伝えること。
「冒険の記録」をコンセプトに、RPGのクエストボードやスキルツリーの要素を取り入れ、採用担当者の記憶に残るサイトを目指す。

### 1.2 デザインコンセプト
* **世界観:** 「風のタクト」×「ニンテンドーDS」ライクなUI。
* **トーン:** 羊皮紙、木目、ドット絵、航海図。
* **UX:** PCでは全体像（地図）を見せ、スマホでは読みやすさ（リスト）を重視する可変レイアウト。

### 2.2 スキルステータス色定義

| クラス名 | 状態 | 背景色 | 枠線色 |

### 2.3 タイポグラフィ
* **メインフォント:** `"M PLUS Rounded 1c", sans-serif`

## 4. レイアウト・レスポンシブ仕様

### 4.1 ブレークポイント
* **768px** を境界とし、PC表示とSP表示を切り替える。

### 4.2 共通ナビゲーション (Bottom Fixed)
* 画面下部に固定配置 (`position: fixed; bottom: 0;`)。
* 4つのメニューボタンを配置。
* カレントページには `.active` クラスを付与し、ハイライト表示（背景色変更＋位置オフセット）。

### 4.3 Aboutページ（スキルマップ）の挙動
本サイトの核心部分。PCとスマホで全く異なるレンダリングを行う。

#### 【PC表示 (769px~)】
* **構造:** CSS Grid (`grid-template-columns: 1fr 1fr 1fr;`)
* **配置ロジック:**
    * **左列:** Security & Mgmt (セキュマネ等)
    * **中列:** Tech Core (基本情報 → 応用情報)
    * **右列:** Infra & Cloud (LinuC → AWS)
* **矢印:** テキスト文字（`↓`, `↘`）をグリッドセルとして配置。
* **合流表現:**
    * 「セキュマネ」の下に `↘` を配置。
    * 「応用情報」の下に `↓` を配置。
    * その合流点に「安全確保支援士」を配置。
    * **安全確保支援士カード:** `grid-column: span 2`（または中央配置）などで視覚的な統合を強調する。

#### 【SP表示 (~768px)】
* **構造:** Flexbox (`flex-direction: column;`)
* **表示順序:** CSS `order` プロパティを使用し、論理的な順序に並べ替える。
    1.  `.sp-order-1`: ITパスポート（始祖）
    2.  `.sp-order-2`: 技術コアルート（基本情報～応用情報）
    3.  `.sp-order-3`: セキュリティルート（セキュマネ～支援士）
    4.  `.sp-order-4`: インフラルート（LinuC～AWS）
* **UI変更:**
    * PC用の矢印セル（`.arrow`, `.arrow-cell`）は `display: none` で非表示。
    * 代わりに各カードの `::before` 擬似要素で簡易的な「↓」を表示。
    * `.mobile-label`（「▼ 技術コア」等の見出し）を表示し、リストを見やすく区切る。

## 5. コンポーネント詳細

### 5.1 カードコンポーネント (`.card`, `.quest-card`, `.work-card`)
* 共通して `border-radius`, `box-shadow`, `border` を持ち、物理的なカード感を演出。
* ホバー時に `transform: translateY(-5px)` で浮き上がるインタラクションを付与。

### 5.2 矢印 (`.arrow`, `.arrow-cell`)
* フォント: `MS PGothic` 等の等幅・記号系フォントを指定し、OSによるズレを防ぐ。
* 色: `--wood` を使用し、不透明度を下げて背景に馴染ませる。

### 5.3 アニメーション (`.fade-up`)
* `script.js` により制御。
* `IntersectionObserver` を使用し、要素が画面内に入ったタイミングで `transform: translateY(0)` と `opacity: 1` を適用。

## 6. 保守・運用ガイド

### 6.1 新しい資格を追加する場合 (`about.html`)
1.  **PC配置:** 所属するカテゴリの列（左・中・右）の適切な行にHTMLを追加する。
2.  **SP順序:** クラス名 `sp-order-X` を付与し、スマホ表示時の並び順を指定する。
3.  **矢印:** 必要に応じて、直前の行に `div.arrow-cell` を追加する。

### 6.2 新しい制作物を追加する場合 (`works.html`)
1.  `.works-grid` 内に新しい `a.work-card` ブロックを追加する。
2.  サムネイルアイコン、タグ、タイトル、説明文を記述する。
3.  グリッドレイアウトのため、自動的に整列される。

### 6.3 配色を変更する場合 (`style.css`)
1.  `:root` 内の変数（`--bg-paper` 等）を変更するだけで、サイト全体の色が一括変更可能。
